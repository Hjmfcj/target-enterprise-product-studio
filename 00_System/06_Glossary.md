# Glossary

## Purpose
Provides a shared vocabulary for terms used consistently across this repository.

## Scope
Covers Product Management, Business Analysis, Solution Architecture, Agile/Scrum, QA, and documentation terminology.

## Intended Usage
| Term | Definition |
|---|---|
| BRD | Business Requirements Document |
| FRD | Functional Requirements Document |
| RTM | Requirements Traceability Matrix |
| MOM | Minutes of Meeting |
| UAT | User Acceptance Testing |
| Epic / Feature / Story / Task | Azure DevOps backlog hierarchy |
| AS-IS / TO-BE | Current-state vs. future-state process models |
| Gap Analysis | Structured comparison identifying missing or incomplete requirements |

*(This glossary grows as new domains and client terminology are onboarded.)*

## Future Expansion
Will expand into a fully cross-referenced, domain-tagged glossary as more projects (pharma, telecom, banking, HR) are onboarded into [10_Projects](../10_Projects/README.md).
