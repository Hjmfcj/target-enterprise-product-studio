# 10_Projects

## Purpose

This directory is the operational workspace for every project managed within the Enterprise Product Studio.

Unlike the other repository sections, which define reusable knowledge, standards, and methodologies, this directory contains the live execution of each individual project.

Every project becomes a self-contained workspace where all project knowledge, decisions, requirements, designs, backlog items, testing artifacts, and delivery outputs are maintained throughout the project lifecycle.

---

## Scope

Applies to every customer engagement regardless of industry, project size, or delivery methodology.

Examples include:

- Pharmaceutical Systems
- Banking Solutions
- Government Projects
- HR Platforms
- PMO Systems
- Internal Products

---

## Repository Philosophy

This folder is not a document repository.

It is a complete Project Workspace.

Each project represents a single source of truth for everything related to that engagement.

The objective is that both humans and AI can understand the complete state of a project by navigating only its workspace.

---

## How to Start a New Project

1. Copy `_Project_Template`.
2. Rename the folder using the approved project naming convention.
3. Complete `Project_Metadata.md`.
4. Complete `Project_Workspace.md`.
5. Begin Discovery.
6. Progress through the remaining project lifecycle.

Never remove template sections.

## Contents

- `_Project_Template`

---

## AI Usage

AI should always begin from the project's **Project_Workspace.md** before reading any other project document.

The workspace provides the current project status, objectives, active phase, open decisions, risks, deliverables, and navigation to all related artifacts.
