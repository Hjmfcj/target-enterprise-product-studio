# Design

## Purpose
Stores solution and UX design artifacts for this project.

## Scope
Applies to this specific project workspace only.

## Intended Usage
Process flows, screen inventories, data models, and architecture notes. Binary/visual artifacts (diagrams, screenshots, mockups, icons) belong in [Assets](Assets/README.md), referenced by relative path rather than embedded.

## Contents
- [Process_Flow.md](Process_Flow.md) — AS-PROPOSED vs. TO-BE flows, BPMN structure, exception flow, gap and conflict analysis
- [Screen_Inventory.md](Screen_Inventory.md) — all screens with evidence, purpose, fields, permissions
- [Data_Model.md](Data_Model.md) — entity list, data dictionary, relationships, CRUD matrix
- [MTN_Travel_System_Figma_Spec.md](MTN_Travel_System_Figma_Spec.md) — production-ready Figma structure, design tokens, component library, screen specifications
- [Assets/](Assets/README.md) — visual artifacts (process flow diagrams, prototype screenshots, mockups, icons)

## Future Expansion
Will accumulate project-specific artifacts as the engagement progresses.
