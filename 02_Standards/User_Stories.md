# User Story Standard

## Purpose
Defines the mandatory format for all User Stories.

## Scope
Applies to all backlog items classified as User Stories in Azure DevOps or equivalent.

## Intended Usage
**Title**: As a [Role], I want [Action], so that [Business Value].

Each story includes: Priority, Dependencies, Business Rules, Validation Rules, and Acceptance Criteria (Given/When/Then).

## Future Expansion
Will be versioned as client-specific variations emerge; deviations must be documented, not silently applied.
