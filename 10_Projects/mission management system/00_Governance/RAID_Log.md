# RAID Log

## Purpose
A running record of Risks, Assumptions, Issues, and Dependencies identified across the entire project, consolidating what would otherwise be scattered across individual BRDs, MOMs, and backlog items.

## Scope
Applies to this specific project workspace only. Covers RAID items from any phase of the engagement. This log is the single source of truth that BRD/FRD Risk, Assumption, and Dependency sections should reference rather than duplicate.

## Intended Usage
Add a new row per item under the relevant category (Risk / Assumption / Issue / Dependency). Do not remove rows once closed — update Status instead. Mark unavailable content as `MISSING INFORMATION`. Per Rule #1 (No Assumptions), any item logged under "Assumption" must be flagged for client confirmation, not treated as settled fact.

**Fields / Sections:** Item ID | Category (Risk/Assumption/Issue/Dependency) | Description | Raised By | Date Raised | Owner | Impact | Mitigation/Response | Status (Open/Mitigated/Closed) | Related BRD/Story Reference

## Future Expansion
Will be refined based on recurring client feedback on deliverable structure.
