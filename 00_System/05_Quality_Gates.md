# Quality Gates

## Purpose
Defines the checks every deliverable must pass before it is considered client-ready, executive-ready, or implementation-ready.

## Scope
Applies before finalizing any BRD, FRD, User Story set, backlog, or process flow.

## Intended Usage
Before finalizing any output, check for:
- Missing requirements
- Missing business rules
- Missing stakeholders
- Missing validations
- Missing approval steps
- Missing integrations
- Missing reports

Every deliverable must conclude with a **Gap Analysis** section summarizing unresolved items.

## Future Expansion
Will be paired with an automated checklist tool once the repository is integrated with a CI/documentation pipeline (see [08_Checklists](../08_Checklists/README.md)).
