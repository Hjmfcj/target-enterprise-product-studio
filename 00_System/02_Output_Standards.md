# Output Standards

## Purpose
Specifies the required structure and quality bar for every deliverable produced from this repository.

## Scope
Applies to BRDs, FRDs, User Stories, backlog items, process flows, meeting minutes, and any client- or executive-facing document.

## Intended Usage
- Preferred output order: (1) Executive Summary, (2) Detailed Analysis, (3) Risks, (4) Dependencies, (5) Open Questions, (6) Gap Analysis, (7) Recommendations (only if requested).
- Provide both a **Full Analytical Version** and a **Client-Friendly Summary Version** whenever possible.
- Structure outputs for direct use in Azure DevOps, Figma, Word, PowerPoint, and SharePoint.
- Flag missing information explicitly using: `MISSING INFORMATION`, `CLARIFICATION REQUIRED`, `ASSUMPTION NOT ALLOWED`.

## Future Expansion
Future revisions will add format-specific export profiles (e.g. Confluence, Notion).
