# Azure DevOps Standard

## Purpose
Defines the backlog hierarchy and required fields for Azure DevOps work items.

## Scope
Applies to all backlog management activity.

## Intended Usage
Hierarchy: Epic → Feature → User Story → Task.

Each User Story includes: Business Value, Acceptance Criteria, Dependencies, Business Rules, and Source Reference. No estimation unless explicitly requested.

## Future Expansion
Will be versioned as client-specific variations emerge; deviations must be documented, not silently applied.
