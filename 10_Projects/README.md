# 10_Projects

## Purpose
Houses per-project workspaces where discovery outputs, BRDs, backlogs, and other artifacts are stored for active engagements.

## Scope
Each active engagement gets its own subfolder, copied from `_Project_Template/`.

## Intended Usage
Copy `_Project_Template/` into a new folder named after the project, then populate it using the templates in [05_Templates](../05_Templates/README.md).

## Contents
- [_Project_Template](_Project_Template/README.md)

## Future Expansion
Will grow to hold one subfolder per active or completed engagement (e.g. pharma, telecom, HR domains).
