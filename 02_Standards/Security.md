# Security Standard

## Purpose
Defines the minimum security-related documentation expected in requirements artifacts.

## Scope
Applies wherever authentication, authorization, or sensitive data handling is in scope.

## Intended Usage
Each security-relevant requirement documents: role-based access implications, data sensitivity classification, and audit/logging needs — flagged as MISSING INFORMATION where undocumented.

## Future Expansion
Will be versioned as client-specific variations emerge; deviations must be documented, not silently applied.
