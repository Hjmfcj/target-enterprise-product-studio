# User Story QA Checklist

## Purpose
Quality checklist to run before finalizing any set of User Stories.

## Scope
Applies immediately before finalizing the matching deliverable type.

## Intended Usage
Work through every item; do not mark complete until all items are addressed or explicitly flagged as MISSING INFORMATION.

**Checklist Items:**
- Title follows As a/I want/So that format
- Acceptance Criteria in Given/When/Then
- Priority, Dependencies, Business Rules, Validation Rules present
- No estimation included unless explicitly requested
- Traceable to an approved FRD/BRD

## Future Expansion
Will be refined based on recurring QA findings across engagements.
