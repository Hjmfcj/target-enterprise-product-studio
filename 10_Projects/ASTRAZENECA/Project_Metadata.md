# Project Metadata

## Purpose
A ready-to-fill, single source of truth for this project's identifying context — most importantly Client and Delivery Company/Branding — so that every downstream document inherits it consistently instead of re-deciding it per file. Directly supports the Branding-Neutral Default rule: branding must never be assumed and should be confirmed here once, from source context, before any client-facing deliverable is produced.

## Scope
Applies to this specific project workspace only.

## Intended Usage
Complete this file first, before populating any phase folder. Do not remove fields — mark unavailable content as `MISSING INFORMATION`. If Branding cannot yet be confirmed from source materials, mark it `CLARIFICATION REQUIRED` rather than defaulting to Target or Uranium Technology Solutions.

**Fields / Sections:** Project Name | Client | Delivery Company | Branding (Target / Uranium Technology Solutions / Client-Specified) | Engagement Type | Current Phase | Product Owner | Business Analyst | Technical Lead | UX Lead | QA Lead | Repository Version | Status
