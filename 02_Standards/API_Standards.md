# API Standards

## Purpose
Defines expectations for documenting integration and API requirements.

## Scope
Applies wherever a BRD/FRD references system integrations.

## Intended Usage
Each integration requirement documents: endpoint/direction, trigger, payload structure (if known), authentication approach, and error handling — flagged as MISSING INFORMATION where undocumented.

## Future Expansion
Will be versioned as client-specific variations emerge; deviations must be documented, not silently applied.
